## **1. 프로젝트 개요**

### **1.1 프로젝트명**

**Founder Sprint Workspace MVP**

### **1.2 한 줄 정의**

> "Batch(기수) 기반으로 질문-피드백-요약이 반복되는 Execution 중심 워크스페이스"

### **1.3 배경**

현재 Founder Sprint 프로그램은 카카오톡, Google Docs, 이메일 등 분산된 도구로 운영되고 있습니다. 이를 하나의 통합된 워크스페이스로 옮겨 운영 효율성을 검증하고자 합니다.

### **1.4 핵심 가치**

- **기수(Batch) 단위 완전 분리**: 각 기수의 데이터와 활동이 독립적으로 관리됨
- **운영 중심 설계**: 커뮤니티/소셜 기능이 아닌, 실행과 피드백에 집중
- **지식 자산 축적**: 질문-답변-요약을 통한 체계적인 지식 아카이빙

## **2. 목적 및 전제**

### **2.1 목적**

- YC Bookface의 기능적 복제가 **아님**
- 향후 외부 SaaS 확장을 전제로 하지 **않음**
- **Outsome / Founder Sprint 내부 전용** 운영 실험용 MVP
- 소수 인원, Batch(기수) 단위 운영 구조 검증

### **2.2 전제 조건**

| **항목** | **내용** |
| --- | --- |
| 사용 범위 | Outsome, Founder Sprint 내부 전용 |
| 운영 규모 | 기수당 Founder 10명 이내, Mentor 10명 정도 |
| 동시 운영 기수 | 1개 (동시에 여러 기수 운영하지 않음) |
| UX/UI 완성도 | 빠른 운영 검증 우선, 완성도보다 기능 동작 중심 |
| 제품 수명 | **6개월 이내 폐기 또는 전면 재작성 가능성** 전제 |

### **2.3 성공 기준**

- 1개 기수(Batch)를 본 시스템으로 완주할 수 있는가?
- 기존 분산 도구(카톡, 이메일, Google Docs) 대비 운영 효율이 개선되는가?

## **4. 핵심 기능 명세**

### **4.1 Batch(기수) 관리**

#### **4.1.1 개요**

모든 데이터는 Batch(기수) 단위로 완전히 분리됩니다. 한 사용자는 하나의 Active Batch에만 속합니다.

#### **4.1.2 Batch 상태**

| **상태** | **설명** |
| --- | --- |
| **Active** | 현재 운영 중인 기수. 모든 기능 사용 가능 |
| **Archived** | 종료된 기수. 전체 Read-only. 열람만 가능, 수정/추가 불가 |

#### **4.1.3 기능 상세**

| **기능** | **설명** |
| --- | --- |
| Batch 생성 | Super Admin/Admin이 새 기수 생성 (이름, 기간, 설명) |
| Batch 종료 | Super Admin/Admin이 수동으로 Archived 상태로 전환 |
| 동시 운영 | **가능** - 여러 Active Batch 동시 운영 가능 |
| Archived 열람 | 해당 기수 참여자 + Admin/Super Admin만 열람 가능 |

#### **4.1.4 Batch 제약 사항**

| **제약** | **설명** |
| --- | --- |
| 생성 제한 | 없음 |
| 삭제 | Super Admin만 가능 (모든 관련 데이터 영구 삭제) |
| 재활성화 | Super Admin이 가능 |
| 이름 | 최대 100자, 중복 허용 |

#### **4.1.5 Batch 정보 항목**

- Batch 이름 (예: "Founder Sprint 5기")
- 시작일
- 종료 예정일
- 설명 (선택, 최대 500자)
- 상태 (Active / Archived)

### **4.2 이벤트 및 오피스아워**

#### **4.2.1 개요**

#### **4.2.2 이벤트 타입**

| **타입** | **설명** |
| --- | --- |
| **일회성 (One-off)** | 단일 일정 이벤트 |
| **오피스아워 (Office Hour)** | Mentor/Admin과 1:1 미팅 |
| **대면 (In-person)** | 오프라인 미팅 |

#### **4.2.3 이벤트 제한**

| **항목** | **내용** |
| --- | --- |
| 최대 횟수 | **20회** (Batch당) |
| 시간대 | **PST / KST** 선택 지원 |
| 시간대 저장 | **UTC** (내부 저장) |

#### **4.2.4 이벤트 생성 (일회성/대면)**

| **항목** | **내용** |
| --- | --- |
| 생성 권한 | Super Admin, Admin |
| 제목 | 필수, 최대 200자 |
| 설명 | 선택, 최대 2,000자 |
| 날짜/시간 | 필수 |
| 장소/링크 | 선택 |
| 대상 | 전체 또는 특정 역할 |

#### **4.2.5 오피스아워 슬롯**

| **항목** | **내용** |
| --- | --- |
| 등록 권한 | Super Admin, Admin, Mentor |
| 슬롯 단위 | **30분 고정** |
| 반복 등록 | 없음 (수동으로 개별 등록) |
| 수정 | 요청 없는 슬롯만 가능 |
| 삭제 | 요청 없는 슬롯만 가능 |
| 과거 시간 | 등록 불가 |

#### **4.2.6 오피스아워 요청 및 승인**

| **항목** | **내용** |
| --- | --- |
| 요청 권한 | Founder, Co-founder |
| 형태 | **1:1** (한 슬롯에 한 명) |
| 다중 요청 | 가능 (Host가 승인할 사람 선택) |
| 승인 권한 | 해당 슬롯의 Host, 또는 Super Admin/Admin |
| 취소 (확정 전) | 요청자가 자유롭게 취소 가능 |
| 취소 (확정 후) | Super Admin/Admin만 취소 가능 |

#### **4.2.7 Google Calendar 연동**

| **항목** | **내용** |
| --- | --- |
| 자동 생성 | 이벤트/오피스아워 승인 시 Google Calendar 일정 자동 생성 |
| 초대 발송 | 참석자 전원에게 Calendar Invite 이메일 발송 |
| Google Meet | 오피스아워 승인 시 자동 생성 |
| **수정/삭제** | 시스템 내 기록 수정, **Google Calendar는 수동 처리 병행** |
| 연동 실패 시 | 에러 메시지 표시 + 수동 처리 안내 |

### **4.3 질문 → 답변 → 요약**

#### **4.3.1 개요**

Founder/Co-founder가 질문을 작성하면, Mentor/Admin이 답변하고, 최종적으로 요약을 작성하여 질문을 마무리합니다.

#### **4.3.2 질문 (Question)**

| **항목** | **내용** |
| --- | --- |
| 작성 권한 | Founder, Co-founder |
| 제목 | 텍스트 (필수, **최대 200자**) |
| 내용 | 텍스트 (필수, **최대 10,000자**) |
| 첨부파일 | 이미지(jpg, png, gif), PDF (**최대 5개, 각 10MB**) |
| 상태 | Open / Closed |
| 수정 | 답변이 달리기 전까지만 가능 |
| 삭제 | 불가 |
| 공개 범위 | 기수 내 전체 공개 (비공개 질문 없음) |

#### **4.3.3 답변 (Answer)**

| **항목** | **내용** |
| --- | --- |
| 작성 권한 | Super Admin, Admin, Mentor |
| 내용 | 텍스트 (**최대 10,000자**) |
| 복수 답변 | 가능 (하나의 질문에 여러 답변) |
| 수정 | 작성자 본인만 가능 |
| 삭제 | **불가** |

#### **4.3.4 요약 (Summary)**

| **항목** | **내용** |
| --- | --- |
| 작성 권한 | Super Admin, Admin |
| 내용 | 텍스트 (**최대 10,000자**) |
| 개수 | 질문 1개당 요약 1개 |
| 효과 | 요약 작성 시 해당 질문은 **Closed** 처리 |
| 수정 | Super Admin/Admin만 가능 |
| 삭제 | Super Admin/Admin만 가능 (요약 삭제 시 질문이 Open 상태로 복귀) |

#### 5.2.2 피드 (Feed) 화면 (#14)
- 레이아웃: 2컬럼 (메인 콘텐츠 ~70% + 우측 사이드바 ~30%)

#### 5.2.3 질문 화면 (#3, #4, #5)
- 레이아웃: 단일 컬럼 (최대 폭 제한)
- 탭 필터: Open / Closed / 전체
- 카드: 제목 + 작성자 + 날짜 + 답변수 + 상태 배지
- 질문 상세: 질문 → 답변 리스트 → 요약 (시간순 나열)

#### 5.2.4 이벤트/오피스아워 (#6, #7, #8)
- 레이아웃: 단일 컬럼
- 탭 필터: 이벤트 / 오피스아워
- 이벤트 생성: 타입 선택 드롭다운 (One-off / Office Hour / In-person)
- 오피스아워 요청: 심플 폼 (드롭다운 + 텍스트 + 타임존)

#### 5.2.5 그룹 화면 (#17, #18, #19)

- 그룹 상세: 피드 화면과 유사한 2컬럼 구조
